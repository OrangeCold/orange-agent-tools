# 字体与排版系统（CJK 可读性适配）

> 选定风格后、生成 HTML 时必读。所有 74 个 DESIGN.md 的排版 token 都是为英文/拉丁字母优化的；当内容是中文（中日韩）时，直接套用会让正文偏小、标题糊成一团。本文件给出 CJK 适配规则——在颜色/圆角/间距/组件仍 100% 忠实品牌的前提下，仅对**字号 / 行高 / 字距 / 字重 / 字体栈**做可读性适配。

## 为什么需要这层适配

中文字符是方块字，没有拉丁字母 a/b/d 的 ascender/descender 来辅助辨识。英文 15px 正文读起来正常，中文 15px 在屏幕上就明显吃力。同理：

- 英文 display 标题用大幅**负字距**（-1.4 ~ -3px）显得紧凑精致，但负字距套到中文标题会让方块字互相叠压、糊成一团。
- 英文用 **300 细体**显得 editorial，中文 300 字重笔画太细、糊眼。
- 英文 fallback 栈（`-apple-system / Segoe UI / Roboto`）不含中文字体，中文会回退到难看的默认 serif。

所以「忠于品牌」不等于「忠于英文 token 的字面像素」——在中文场景，忠于品牌的**可读性意图**才对。

## 触发判断

检测正文是否含 **CJK 字符**（中日韩统一表意文字 U+4E00–U+9FFF，含日文假名、韩文谚文区间）：

- **含 CJK** → 全页排版维度统一按下方 CJK 规则处理（即便夹杂英文标题，也避免负字距，保持页面一致性）。
- **纯英文/拉丁内容** → 保持 DESIGN.md 原 token 不变，本规则不介入。
- **混合内容** → 正文含 CJK 即按 CJK 取严。

## CJK 排版规则

| 维度 | DESIGN.md 英文 token | CJK 适配（覆盖原值） |
|---|---|---|
| 正文最小 | 常见 15–16px | **≥16px**；长文 / 周报推荐 **17–18px** |
| 辅助文字红线 | caption 12–13px，micro 低至 10px | CJK caption **≥14px**；法律极小字 ≥12px；**禁止 10–11px** 出现在任何可读文本 |
| 行高 | 英文 1.4–1.5 | CJK 正文 **1.7–1.8**（长文可达 2.0）；标题 1.3–1.4 |
| 字距 | display 负 tracking（-1.4 ~ -3px） | **禁用负字距**；标题 0 ~ +0.05em；正文 0 |
| 字重 | 部分（如 Stripe）全 300 细体 | CJK 正文 **≥400**，禁用 <400；标题 500–700 |
| 行宽 | 英文 45–75 字符 | CJK 每行 **20–35 字**（用 `max-width` 控制） |
| 移动端 | 多数 DESIGN.md 移动端缩小字号 | CJK 正文 / 导航 / 按钮**不缩小**，保持 ≥16px（仅大标题 display 可缩） |

## 字体栈（CJK fallback）

DESIGN.md 给的字体栈通常是纯英文。生成中文内容时，在英文字体之后补上中文字体作 fallback（英文优先用英文显示字体，中文字符回退到中文字体）：

```html
font-family: 'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto,
  'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Noto Sans SC', sans-serif;
```

- **优先用系统字体**：PingFang SC（macOS/iOS）、Microsoft YaHei（Windows）、Hiragino Sans GB——零体积、零首屏延迟，且观感优秀。
- **不强制引 Web 中文字体**：Noto Sans SC 等体积大、拖慢首屏。仅当风格强要求品牌一致、且用户明确需要时，按需引对应字重的 Google Fonts。

等宽字体（代码）同理补 `'PingFang SC'` 等，保证代码里的中文注释也清晰。

## 与品牌忠实的关系

- **仍 100% 忠实**：颜色 hex、圆角、间距、组件形态、hover/pressed 状态——全部用 DESIGN.md 原 token，不自创。
- **仅这 5 个维度适配**：字号、行高、字距、字重、字体栈——且只在 CJK 内容时覆盖。这是「忠于品牌的可读性意图」，不是破坏品牌识别度。
