# 风格索引（74 个真实品牌设计系统）

这是 design-to-html 的「目录」。**永远不要一次性读完 `design-md/` 下 74 个文件**——先读本表，按用户内容匹配出 1–3 个候选，再只读选中那一个的 `references/design-md/<slug>.md`。

## 如何匹配

读懂用户内容后，沿 4 个维度判断，在下方表中找最契合的 1–3 行：

1. **内容类型**：产品落地页 / 文章长文 / 简历作品集 / 数据看板 / 营销站 / 复古趣味页…
2. **调性氛围**：极简精密 / 暖编辑感 / 暗黑酷炫 / 奢华高端 / 活泼友好 / 技术蓝图 / 复古…
3. **明暗偏好**：用户没明说就按内容推断（技术·游戏·创意工具常偏暗；编辑·温暖·零售常偏亮）
4. **行业线索**：金融 / 汽车 / AI / 电商 / 媒体 等关键词直接指向对应行业品牌

命中唯一即用；多个都说得通就用选择题让用户选。用户点名某个品牌（如「做成 Stripe 那种」）时直接锁定。

## 列含义

- **读取**：匹配后读取的文件 = `references/design-md/<读取>.md`
- **明暗**：页面主底色基调（light / dark；少数品牌混用会标注）
- **强调色**：取自该 DESIGN.md 的真实主色，仅作调性提示——**生成时一律以文件内的精确 hex 为准**
