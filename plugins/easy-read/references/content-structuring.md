# 内容结构化准则（金字塔原理）

> easy-read 两个 skill（html / feishu）共享的前置逻辑。在"生成产物"之前，先用金字塔原理把用户的原始内容重排为清晰的层级结构。

## 核心目标

让内容**更易读、更易理解**：读者第一眼看到最重要的结论，往下逐层看到支撑，且每层内部逻辑清晰、不重不漏。

## 金字塔四步

> 方法论依据：金字塔原理（Barbara Minto）的“结论先行”与 NN/g 的倒金字塔结构、美国《Plain Writing Act》的“要点先行”原则一致——都是把最重要的结论置顶，细节按重要性递减，让扫读者第一眼抓到主旨、可从底部裁剪而不丢要点。

1. **结论先行** —— 把最重要的结论放在最顶层。结论须明确、可独立理解。
2. **以上统下** —— 上层结论必须能概括其下所有内容；下层全部都是上层结论的支撑，没有多余、没有缺失。
3. **归类分组** —— 同一层的要点按逻辑关系分组，组内遵循 MECE（相互独立、完全穷尽）。
4. **逻辑递进** —— 组内要点按 时间 / 结构 / 重要性 之一排列，整组呈现递进。

## 输出骨架

```
核心结论（一句话，最重要）
├── 支撑要点 A
│   ├── 论据/细节 A1
│   └── 论据/细节 A2
├── 支撑要点 B
│   └── ...
└── 支撑要点 C
```

## 忠实性原则（硬约束）

- **只重排，不改写、不补造**。重构 = 调整顺序、显化隐含逻辑、压缩冗余，不是改写措辞或扩写。
- 原文明示的事实/结论才可写入结构；原文支撑不了的，**不**自行补全，归入 SKILL.md 的"澄清"步骤用选择题问用户。
- 可做的：合并重复点、把散落的同一主题归到一组、把隐含的因果显化为“因为…所以…”、删除与结论无关的冗余。
- 不可做的：替用户下结论、补造数据/事例、改变原意、把“可能”说成“一定”。

### 改写边界（重排 vs 改写的具体判定）

“只重排、不改写”是硬约束，但重排与改写的边界需明确，避免执行时摇摆：

| ✅ 允许（重排，忠实） | ❌ 禁止（改写，违背忠实） |
|---|---|
| 合并重复表述、散落同主题归组 | 替换同义词、改变措辞风格 |
| 长复合句的并列分句拆成列表多个 item（措辞基本不变） | 被动语态改主动语态 |
| 隐含因果显化为“因为…所以…” | 补原文没有的连接词 / 过渡句 |
| 删除与结论无关的冗余 | 替用户下结论、补造数据 / 事例 |
| 长句断句（保原词，仅切分标点 / 拆 item） | 把“可能 / 预计”说成“一定 / 确定” |
| 提取原文已有要点为小标题 | 改变原意、夸大或弱化程度 |

判定原则：**措辞是否基本保留**。只要原文的词、语气、确定性被保留，仅调顺序 / 分组 / 层级 / 切分，属重排；一旦动措辞、语气或确定性，就是改写——回 SKILL.md 澄清步骤问用户。

## 分块（chunking）

工作记忆同时只能处理约 4 个信息块（Cowan 2001；经典 Miller 1956 为 7±2）。骨架每一层都要控制并列要点数量：

- **同层并列要点 ≤4 个**为理想；超过时优先**归并为更少的组**，或**下沉一层**（拆 H3 / 子组），不要平铺 7–8 个并列项。
- 一个信息块 = 一个可独立理解的单元（一个论点 / 一个步骤 / 一组对比），不是机械按句数切。
- 分块不仅是视觉分隔，更是**显化内容间关系与层级**的手段——相关要点归组、用组标题点明关系。

## 术语一致性

- **首次出现释义**：专业术语 / 缩写首次出现时显化含义（原文已有解释直接用；没有则归入澄清步骤问用户，不编造）。
- **全文统一用词**：同一概念全文用同一个词，不要前半篇“创作者”后半篇“主播”交替（选原文更常用或更正式的）。
- **术语对照表**：术语密集的文档（系统设计 / 技术方案），末尾用表格给“术语 → 释义”对照。
- 这是**显化原文已有信息**（统一用词、提取已有释义），不是补造——仍受忠实性约束。

## 与图表化的衔接

结构化产出骨架后，识别其中适合图示的部分（关系/顺序/结构/时间线），交由 `diagram-guide.md` 选型画图；不适合图示的（纯结论、对比、清单）用卡片/表格/列表。结论本身始终以醒目的文字呈现（如 hero 大标题）。
