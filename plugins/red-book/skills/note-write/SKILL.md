# note-write

把产品/主题写成**真实、不浮夸、有场景感**的小红书笔记。

核心 = 真人分享口吻 + 清晰结构 + 具体场景。不是硬广，是"用过的人在分享"。

## 为什么这样设计

小红书用户对硬广和浮夸词（神器 / yyds / 封神 / 绝绝子）已免疫，真实、具体、有共鸣的分享反而更打动人。所以本 skill 的命脉是**像真人分享**：第一人称、口语、有具体场景和细节，而不是堆砌功能形容词。

## 硬约束（必须遵守）

- **标题 ≤ 20 字**：小红书硬限制，标点和 emoji 都算字符，留 1-2 字余量更稳。
- **不浮夸**：禁用 神器 / yyds / 封神 / 绝绝子 / 秒杀 / 直接起飞 / 提升 N 倍 这类词。
- **第一人称**：用"我"，不说教（"你值得拥有"这种禁用）。
- **口语化**：短句、短段，像聊天。
- **emoji 适度**：用于分隔和点缀，不堆砌。

## 工作流（6 步）

### 第 1 步：理解要写什么

判断：产品/主题是什么、核心卖点 1-3 个、目标受众（决定用词与场景）、必带信息（地址/链接/价格/活动）。

### 第 2 步：拟标题（≤20 字）

- 突出核心关键词（产品名 / 核心能力 / 核心结果）。
- 给 3-5 个不同风格备选，推荐一个，让用户定。
- 风格池见下表。
- 自检字数 ≤ 20。

### 第 3 步：功能介绍（第一部分）

**直接说"它干什么"**，不绕弯子讲故事开头。一句话定位 + 分点说核心能力（2-4 点）。用"把 X 变成 Y"这种具体转化，不用抽象口号。

### 第 4 步：真实场景（第二部分，重点）

列 3-4 个场景，每个三段式：

- **痛点现象**：具体、有共鸣（"开完两小时会，记了一屏流水账"）。
- **怎么用**：丢进去 / 扔给它（动作具体）。
- **效果**：可感知的改变（"发群里没人再问然后呢"）。

场景要真实可信，带数字和细节，不堆形容词。**最大众的痛点放第一个**。

### 第 5 步：用心细节 + 收尾

- 3-4 条"用心的细节"，体现真的用过、有打磨。
- 收尾：一句平实总结 + 引导互动（地址/链接放评论，避免限流）+ 标签 5-8 个（蹭相关热点话题）。

### 第 6 步：自检

逐条过：

- 标题字数 ≤ 20？
- 浮夸词逐个排查？
- 第一部分是功能介绍（不是故事）？
- 场景够具体（有数字/细节/动作）？
- 第一人称口语？
- 必带信息齐全？

## 标题备选风格池

| 风格 | 写法 |
|------|------|
| 直给 | 把 X 变成 Y |
| 口语 | 丢 X 进去，出来 Y |
| 简短 | X 变 Y |
| 结果导向 | 一段 X，还你 Y |
| 走心共鸣 | 情绪 + 转折 |
| 好奇钩子 | 悬念 |
| 冲突反转 | 冲突 + 悟了 |
| 场景直给 | 场景？我有招 |

## 场景三段式范例

> 📌 **AI 写的方案，读不下去**
> 让 AI 写份方案，出来一大坨平铺文字，逻辑在但重点不突出、没图，发出去没人想看（痛点）。把这段丢进去（动作），它理成结论先行的结构、把关键流程画成图、套个品牌配色，变成一份能直接发的网页（效果）。

## 关键原则

- **真实 > 浮夸**：像用过的人在分享，不像广告。
- **具体场景 > 抽象功能**：用场景讲功能，不堆功能清单。
- **口语 > 书面**：短句聊天感。
- **第一部分讲功能，第二部分讲场景**：用户要的明确结构。
- **标题突出核心关键词 + ≤20 字**：硬限制。
